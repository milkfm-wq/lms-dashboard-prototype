## 7) 🟠 Phase D — Pre-Production (In Progress)

Phase D разделена на отдельные треки, чтобы не смешивать разнонаправленные задачи.

### 🟠 D1. DB Migration Track (Supabase)

**Цель:** перевести БД из локального режима в managed Postgres (Supabase) для staging.

**Scope:**
- Создать/проверить проект Supabase.
- Настроить `DATABASE_URL` и `DIRECT_DATABASE_URL`.
- Применить миграции в целевой БД.
- Настроить резервное копирование и тест восстановления.
- Проверить smoke для критичных read/write API.

**DoD:**
- миграции применяются без drift;
- backup/restore тест успешен;
- API smoke на Supabase проходит.

### 🟡 D2. App Deploy Track (Vercel)

**Цель:** вынести приложение онлайн на staging-домен.

**Scope:**
- Подключить репозиторий в Vercel.
- Настроить env/secrets.
- Проверить build/runtime в staging.
- Настроить домен (или staging URL).
- Прогнать post-deploy smoke.

**DoD:**
- staging доступен онлайн;
- auth и защищённые API работают;
- критичные экраны и импорт read-потоков проходят smoke.

### 🟡 D3. Security & Ops Track

**Scope:**
- Политика паролей/сессий для staging.
- Базовый мониторинг ошибок и журналов.
- Чеклист инцидентных действий (rollback/restore).

### 🟡 D4. Release Readiness Track

**Scope:**
- Финальный pre-prod checklist.
- Набор тестовых пользователей `mentor/assistant/student`.
- End-to-end прогон перед soft launch.
