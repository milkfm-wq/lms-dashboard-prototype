# Dashboard Spec v1

Статус: Draft  
Дата: 2026-03-30  
Владелец: mentor + product/engineering

## 1. Объём v1

- Фокус: дашборд наставника (`mentor`).
- Формат: мокап/прототип без привязки к БД.
- Цель: согласовать состав блоков, визуальную структуру и UX до интеграции данных.

## 2. Макет экрана (верхний уровень)

- Header: название потока, текущая неделя, дата последнего обновления.
- Filter bar (`F*`): период, неделя, статус участника.
- KPI row (`M*`): 4 карточки в одну линию на desktop.
- Trends row (`C*`): 2 графика рядом.
- Main table (`T1`): таблица участников с быстрыми статусами.
- Side panel (`S1`): карточка выбранного ученика (drill-down из `T1`).

### 2.1. Wireframe (черновой)

1. Header  
2. Filter bar (`F1/F2/F3`)  
3. KPI cards (`M1/M2/M3/M4`)  
4. Charts (`C1/C2`)  
5. Table (`T1`) + right panel (`S1`)

## 4. Описание блоков

### F1 — Период
- Назначение: управляет временным окном для KPI/графиков/таблицы.
- Значения: `7 дней`, `30 дней`, `90 дней`, `текущий поток`.
- Поведение: изменение фильтра перерисовывает весь экран.

### F2 — Неделя потока
- Назначение: быстрый фокус на конкретную учебную неделю.
- Значения: `all` + `week 1..9`.
- Поведение: при выборе недели приоритет над `F1` для недельных блоков.

### F3 — Статус участника
- Назначение: фильтр таблицы и агрегатов по активности.
- Значения: `все`, `активные`, `выбывшие`.

### M1 — Общий прогресс группы
- Назначение: главная контрольная метрика состояния потока.
- Что показываем: процент `0..100` + delta к прошлой неделе.
- Формула: среднее по `progress_percent` участников (уточняется в Data Contract).
- Drill-down: клик ведёт к `C1`.
- Empty state: `Недостаточно данных для расчёта`.

### M2 — Средний рейтинг
- Назначение: агрегированная оценка динамики группы.
- Что показываем: средний рейтинг + стрелка тренда.
- Формула: среднее по rating (точная формула уточняется).

### M3 — В зоне риска
- Назначение: оперативно показать критичные случаи.
- Что показываем: число участников `risk`.
- Drill-down: открывает предфильтрованную `T1` (`status=risk`).

### M4 — Лидеры недели
- Назначение: выделить позитивную динамику и мотивацию.
- Что показываем: число лидеров + top-3 имён в tooltip/mini-list.

### T1 — Таблица участников
- Колонки: `Ученик`, `Проект`, `Прогресс %`, `Рейтинг`, `Дисциплина`, `Риск`, `Лидер`, `Статус`.
- Сортировки: по прогрессу, рейтингу, дисциплине, имени.
- Действия: выбор строки открывает `S1`; быстрый фильтр `только риск`.
- Drill-down в `S1`: клик по строке показывает детальную карточку.

### C1 — Динамика прогресса группы
- Тип графика: line chart.
- Ось X: недели/дни (зависит от `F1`).
- Ось Y: `прогресс %`.
- Цель: увидеть тренд потока.

### C2 — Дисциплина по неделям
- Тип графика: bar/stacked bar.
- Ось X: недели.
- Ось Y: индекс дисциплины / количество нарушений.
- Цель: быстро определить проблемные недели.

### S1 — Карточка ученика (правая панель)
- Состав: проект, точка A/B, прогресс, рейтинг, дисциплина, задачи, штрафы, история по неделям.
- Источник открытия: только из `T1`.
- Режим: read-only в мокапе.

## 5. Визуальные требования (wow-effect)

- Плотный, премиальный layout без “админки по умолчанию”.
- Яркая иерархия: KPI -> тренды -> детализация.
- Цветовые коды статусов:
  - risk: `red`,
  - leader: `green`,
  - neutral: `slate`.
- Читаемость на ноутбуке и проекторе (контраст + крупные цифры в KPI).
- Обязательный эффект: “одним взглядом понятно состояние группы”.

## 6. Out of scope v1

- Реальные данные из БД.
- Финальная роль-дифференциация assistant/student.
- Экспорт PDF.
- Финальные формулы KPI (фиксируются после интервью с наставником).
